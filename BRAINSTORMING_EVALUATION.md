## 📊 현재 모듈 구조 요약

### 핵심 특징:
1. **Ephemeral RAG** (세션별 임시 벡터 DB, 자동 삭제)
2. **영구 RAG** (SCAMPER, Mind Mapping, Starbursting 기법)
3. **직군 추론** (키워드 기반 domain hints)
4. **3단계 플로우** (Q1: 목적 → Q2: 워밍업 → Q3: 자유연상 30초)
5. **SWOT 분석** (아이디어별 강점/약점/기회/위협)

### 프롬프트 철학:
- "허구 데이터 절대 금지" (통계, 비용, 경쟁사 실적 지어내지 않기)
- "현실적 실행 가능성" (사용자가 바로 시작 가능한 것만)
- "직군별 맞춤" (유튜버, 소상공인, 개발자 등)

## ⚠️ 약점 (개선 필요)

### 1. **RAG 활용이 표면적** ⭐⭐☆☆☆ (치명적)

#### 현재 문제:
```python
# 영구 RAG: 상위 3개만 검색, 500자만 사용
techniques_results = self._search_permanent_rag(query=purpose, n_results=3)
techniques_str = "\n\n".join([
    f"{t['content'][:500]}..."  # ❌ 500자 자르기
])

# 프롬프트에서 "참고하세요" 정도만
prompt = f"""【적용 가능한 브레인스토밍 기법】
{techniques_str}

위 규칙을 엄격히 지키며...
"""
```

#### 문제점:
1. **RAG 결과를 "참고만" 함**
   - "SCAMPER의 Substitute 기법을 반드시 적용" 같은 강제성 없음
   - LLM이 RAG 무시하고 독자적으로 생성할 가능성 큼

2. **500자는 너무 적음**
   - SCAMPER 전체 설명이 1000자 이상인데 절반만 사용
   - 핵심 내용이 잘릴 가능성

3. **Top-3은 적음**
   - 일반적으로 RAG는 top_k=5~10
   - 다양성 부족

#### 개선 방안:
```python
# ✅ 개선 후
techniques_results = self._search_permanent_rag(query=purpose, n_results=5)
techniques_str = "\n\n".join([t['content']])  # 전체 사용

prompt = f"""【필수 적용 기법】
아래 브레인스토밍 기법 중 **최소 2개 이상**을 명시적으로 적용하세요.
각 아이디어에 "적용된 기법: SCAMPER - Substitute" 같은 식으로 표시하세요.

{techniques_str}

**🚨 규칙**: 위 기법을 사용하지 않으면 답변 무효입니다.
"""
```

#### 측정 지표:
- **RAG 기여도**: 아이디어에 RAG 기법이 실제로 적용되었는가?
- **인용율**: "SCAMPER", "Mind Mapping" 같은 기법명이 답변에 몇 번 등장?

### 2. **Ephemeral RAG의 논리적 효용성 의문** ⭐⭐⭐☆☆

#### 현재 로직:
```python
# Q1(목적)과 Q3(키워드) 유사도로 상위 7개 추출
keywords = ephemeral_rag.extract_keywords_by_similarity(purpose, top_k=7)
```

#### 문제:
- Q1: "소상공인 홍보 앱"
- Q3: ["인스타", "네이버", "블로그", "할인", "이벤트", ...]
- **→ 당연히 모두 유사함** (필터링 효과 거의 없음)

#### 비용-효과 분석:
- **비용**: 20개 키워드 임베딩 + ChromaDB 저장/삭제
- **효과**: 상위 7개 추출 (실제로는 순서만 바뀜)
- **→ 차라리 Q3 전체를 프롬프트에 직접 삽입하는 게 더 효율적**

#### 개선 방안 (2가지):

**Option 1: Ephemeral RAG 폐기**
```python
# 단순화
keyword_str = ", ".join(associations)  # Q3 전체 사용
```
- 장점: 구현 간단, 비용 절감
- 단점: "RAG 활용" 스토리 약해짐

**Option 2: Ephemeral RAG 고도화**
```python
# 다양한 검색 전략
def search_by_diversity(self, purpose, keywords):
    """
    1. 목적과 유사한 키워드 (상위 3개)
    2. 목적과 다른 키워드 (하위 3개) → 창의성
    3. 중간 유사도 키워드 (중간 3개) → 균형
    """
```
- 장점: "다양성 확보" 논리, RAG 의미 강화
- 단점: 구현 복잡

#### 추천:
- **Option 2** (RAG 고도화)
- 발표에서 "단순 유사도가 아니라 다양성도 고려" 스토리

### 3. **창의성 억제** ⭐⭐☆☆☆

#### 현재 프롬프트:
```python
**나쁜 예 (절대 금지)**
- "글로벌 시장 진출 전략..."
- "대형 투자사 IR..."
- "유명인 섭외..."
```

#### 문제:
- 브레인스토밍은 **발산적 사고**가 핵심
- 지금은 너무 **수렴적** (안전한 아이디어만)
- "일론 머스크 섭외"도 브레인스토밍 단계에선 OK
  - 실행 단계에서 걸러내면 됨

#### 딜레마:
| 창의성 높임 | vs | 실용성 높임 |
|-------------|-----|-------------|
| "화성 여행 앱" | vs | "네이버 블로그 만들기" |
| ✅ 재미있음 | vs | ✅ 실용적 |
| ❌ 비현실적 | vs | ❌ 창의성 낮음 |

#### 개선 방안 (단계별 접근):

**Phase 1: 발산 (창의성 최대화)**
```python
prompt_phase1 = """
목적: {purpose}

**규칙**: 
- 제약 없이 자유롭게 상상하세요
- 비현실적이어도 OK
- 10개 아이디어 생성

예: "일론 머스크와 협업", "AI가 자동으로 영상 편집", "드론 배달"
"""
```

**Phase 2: 수렴 (실용성 필터링)**
```python
prompt_phase2 = """
위 10개 아이디어 중:
- 빠르게 시작 가능한 3개 선택
- 각각을 현실적으로 수정

예:
- "일론 머스크 협업" → "업계 선배에게 조언 구하기"
- "AI 자동 편집" → "캡컷 무료 템플릿 활용"
"""
```

#### 측정 지표:
- **창의성 점수**: LLM Judge가 평가 (1-10점)
- **실용성 점수**: LLM Judge가 평가 (1-10점)
- **목표**: 창의성 7점 이상, 실용성 8점 이상

## 💡 창의성 vs 실용성 딜레마 해결 전략

### **현재 접근 (수렴 중심)**:
```
창의성 ━━━━━━━━━━━━━━━━━━━━━━━━━━ 실용성
        ↑ (지금 여기)
```
- 안전하지만 지루함
- "네이버 블로그 만들기" 수준

---

### **개선 접근 (단계별 분리)**:
```
Phase 1 (발산):
창의성 ━━━━━━━━━━━━━━━━━━━━━━━━━━ 실용성
                            ↑ (제약 없음)

Phase 2 (수렴):
창의성 ━━━━━━━━━━━━━━━━━━━━━━━━━━ 실용성
        ↑ (현실적으로 조정)
```

---

### **구체적 예시**:

#### Q1: "유튜버 콘텐츠 아이디어"

**Phase 1 (창의성 우선)**:
1. "일론 머스크 인터뷰"
2. "AI가 자동으로 영상 편집"
3. "구독자와 화성 여행"
4. "뇌파 측정 콘텐츠"
